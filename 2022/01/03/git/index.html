基础篇

git commint

解释

创建一个新的提交记录

图解

git commit

git branch

解释

创建一个新的分支

图解

git branch bugFix

git checkout bugFix

简化命令：git checkout -b <new branch name> 创建一个新的分支同时切换到新的分支

git merge

解释

合并分支。例如：有两个分支AB，现在在A分支上面，想把A分支的修改合并到B分支上面，就需要先切换到B分支上，然后进行git merge A

图解

git branch bugFix

git checkout bugFix

git commit

git checkout main

git commit

git merge bugFix

git rebase

解释

取出一系列的提交记录，“复制”他们，然后在另一个地方逐个放下去

图解

git checkout -b bugFix

git commit

git checkout main

git commit

git checkout bugFix

git rebase main

高级篇

git head

解释

指向某次提交记录

图解

git checkout c4

相对引用

解释

  • 使用^向上移动一个提交记录
  • 使用~<num>向上移动多个提交记录

图解

git checkout c4

git checkout HEAD^

相对引用2

解释

使用~<num>命令来让分支向上移动多次

图解

git branch -f main c6

git branch -f <branch name> <commint name> 强制命令，强制将分支指向某次提交记录

git checkout HEAD^

git branch -f bugFix HEAD~1

撤销变更

解释

  • git reset: 通过把分支回退几个提交记录来实现撤销改动（只会修改本地的改动，对远程没有修改）
  • git revert: 和git reset命令作用相同，但是它会同步修改远程的提交记录

图解

git reset HEAD~1

git checkout pushed; git revert HEAD

因为c2’的更改是撤销c2的所以c2’的状态和c1是相同的

移动提交记录

git cherry-pick <提交号>

解释

将一些提交放到当前位置下面

图解

git cherry-pick c3 c4 c7

交互式的rebase

解释

git rebase -i branch/HEAD~<num>可以打开一个UI界面：

  • 调整提交记录的顺序<鼠标拖拽完成>
  • 删除不想要的提交记录
  • 合并提交

图解

git rebase -i HEAD~4

杂项

本地栈式提交

解释

当程序遇到棘手bug时我们会尝试在程序中加入日志发现bug问题所在，当我们解决问题后将bugFix分支的代码合并到main分支时那些调试的日志也会携带过去，当然这并不是我们想要的，所以我们可以通过git rebase -i 和 git cherry-pick命令将最后的提交记录合并到主分支即可

图解

git rebase -i HEAD~3

git checkout main; git branch -f main c4'

提交技巧1

解释

你之前在 newImage 分支上进行了一次提交，然后又基于它创建了 caption 分支，然后又提交了一次。此时你想对某个以前的提交记录进行一些小小的调整。比如设计师想修改一下 newImage 中图片的分辨率，尽管那个提交记录并不是最新的了。

图解

git rebase -i HEAD~2

git commit --amend

git rebase -i HEAD~2

git branch -f main c3''

提交技巧2

解释

使用git cherry-pick来解决提交技巧1中的问题

图解

git checkout main; git cherry-pick c2

git commit --amend

git cherry-pick c3

git tag

解释

在某次提交上面做一个标记

图解

git tag v0 c1

git tag v1 c2; git checkout c2

git describe

解释

git tab会在提交记录上面做一个标记，而git describe <ref>会描述距离ref最近的tag的多少提交记录，如果没有指定ref那么会在你所在的位置做过检出点

图解

git describe bugFix

多次rebase

git rebase main bugFix

git rebase bugFix side

git rebase side another
