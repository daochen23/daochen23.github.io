基础

  1. 面向对象编程有哪些特征？

    • 封装：封装就是隐藏对象的属性和实现细节，仅对外公开接口，控制在程序中属性的读和修改的访问级别，将抽象得到的数据和行为相结合，，也就是将数据与操作数据的源代码进行有机整合，形成类，其中数据和方法都是类的成员
    • **继承：**继承是面向对象的基本特征之一，继承机制允许创建分等级层次的类。
      继承就是子类继承父类的属性和行为，使得子类对象（实例）具有父类的实例和方法，或子类从父类继承方法，使得子类具有父类相同的行为。
    • **多态：**多态同一个行为具有多个不同的表现形式或形态的能力，是指一个类的实例的相同方法在不同情形有不同表现形式。多态的机制使具有不同内部结构的对象可以共享相同的外部接口，这意味着，虽然针对不同对象的具体操作不同，但是通过一个公共类，它们（那些操作）可以通过相同的方式予以调用。

  2. JDK 与 JRE 的区别是什么？

    • JRE（Java Runtime Enviroment）：JRE 是 Java 的运行环境，面向 Java 程序的使用者，而不是开发者。仅下载并安装了 JRE ，那么系统只能运行 Java 程序。
      JRE 是运行 Java 程序所必须环境的集合，包含 JVM 标准实现及 Java 核心类库。它包括 Java 虚拟机、Java 平台核心类和支持文件。它不包含开发工具（编译器、调试器等）。
    • JDK（Java Developement Kit）: 是 Java 开发工具包，它提供了 Java 的开发环境（提供了编译器 javac 等工具，用于将 java 文件编译为 class 文件）和运行环境（提供了 JVM 和 Runtime 辅助包，用于解析 class 文件使其得到运行）。
      下载并安装了 JDK ,那么不仅可以开发 Java 程序，也同时拥有了运行 Java 程序的平台。
      JDK 是整个 Java 的核心，包括了 Java 运行环境（JRE），一堆 Java 工具包和 Java 标准类库。

  5. == 和 equals 比较有什么区别？

    • ==：

      如果比较的对象是基本数据类型，则比较的是数值是否一致；

      如果比较的是引用数据类型，则比较的是对象的地址是否一致；

    • equals():

      equals() 默认用来比较对象的地址是否一致，不能用于比较基本数据类型，如果对象和自身作比较，则 equals() 方法与 == 的作用是一样的。

      为什么常用 equals() 来比较 String 字符串的内容相等？

      因为对于String、Date、Integer等类型重写了 equals() 方法，使其比较的是存储对象的内容是否相等，而不是堆内存地址。

      public boolean equals(Object anObject) {
    if (this == anObject) {
        return true;
    }
    if (anObject instanceof String) {
        String anotherString = (String)anObject;
        int n = value.length;
        if (n == anotherString.value.length) {
            char v1[] = value;
            char v2[] = anotherString.value;
            int i = 0;
            while (n-- != 0) {
                if (v1[i] != v2[i])
                    return false;
                i++;
            }
            return true;
        }
    }
    return false;
}

  7. this 和 super 有什么区别？

    this：代表当前对象本身；

    this(…): 调用本类的构造方法；

    this.成员变量：调用当前对象的成员变量；

    this.成员方法(…)：调用本类的成员方法；

    super：代表当前 对象的父类；

    super(…)：调用父类的构造方法；

    super.成员变量：调用父类的的成员变量；

    super.成员方法(…)：调用父类的成员方法；

  10. float n = 1.8 有错吗？

    有错。1.8 默认是双精度(double)类型，将double类型赋值给 float 类型属于下转型（down-casting，也称为窄化）会造成精度损失，因此需要强制类型转换：

    float n = (float)1.8 或者 float n = 1.8F

  11. i++ 和 ++i 的区别？

    i++ 是先赋值再加， ++i 是先加再赋值。小技巧：符号在前就先加/减，符号在后就后加/减

  12. while 和 do while 有啥区别？

    while 和 do while 都是循环判断语句，不同的是 while 是先判断条件再执行循环，而 do while 是先执行循环在判断条件，所以说，do while 循环至少会执行一次。

  13. 如何跳出 Java 中的循环？

    可以使用如下关键字：

    • **continue：**跳出本次循环；
    • **break：**跳出整个循环；
    • **return：**跳出整个循环及当前方法；

  14. 如何跳出 Java 中的多层嵌套循环？

    可以在循环外面定义一个无特殊意义的符号，然后在最里层循环体中使用 break 符号 语句跳出来嵌套循环，如：

    @Test
public void test09() {
    jumpLoop:
    for (int i = 0; i < 9; i++) {
        for (int j = 0; j < i; j++) {
            System.out.print("这是 j=" + j);
            break jumpLoop;
        }
        System.out.println("这是 i=" + i);
    }
}

    注意：特殊符号后面是冒号不是分号。

  15. & 和 && 的区别？

    & 和 && 都是可以作为逻辑运算符，表示逻辑与（and），当运算符两边的表达式的结果都为 true 时，整个表达式运算结果才 为true，否则，只要有一方为 false，则结果就是 false。

    && 还具有短路功能，如果第一个表达式为 false ，则不再计算第二个表达式，例如 if(str != null && !str.equals(""))当 str 为null时，后面的表达式是不会执行的，所以是不会出现空指针异常的，但是当使用 & 连接两个表达式的时候就会出现空指针异常。

    & 还可以作为位运算符，当 & 操作符两边都不是 boolean 类型时，& 表示按位与操作。

  16. 2 * 8 最有效率的计算方法是什么 ？

    2 << 3 左移 3 位相当于乘以 2 的 3 次方，右移 3 位相当于除以 2 的 3 次方。因为位运算 cup 是直接支持的，效率是最高的

  17. 数组有没有 length 方法？String 呢？

    数组没有 length() 方法，但是有 length 属性。

    String 有 length() 方法。

  18. 怎么理解值传递和引用传递？

    • 值传递：

      传递的是基本类型参数的字面量值的拷贝，方法对参数的修改不会影响之前的参数值。

    • 引用传递：

      传递的是该参数所引用的对象在堆中的地址值的拷贝，方法对参数的修改会直接影响参数之前的值。

  19. Java 到底是值传递还是引用传递？

    Java 中方法参数严格来说是按照值传递的。

    如果参数是基本数据类型：传递的是基本数据类型的字面值的拷贝。

    如果参数是引用数据类型：传递的是该参数所引用的对象在堆中地址值的拷贝。

  20. 一个 “.java” 源文件的类有什么限制？

    一个 .java 源文件可以创建多个 Java 类，但最多只能创建一个公开类( public class), 而且文件名必须和公开类名保持一致。

  21. Java 中的注释有哪些写法？

    1. 单行注释： //......
    2. 块注释：/*......*/
    3. 文档注释：/**......*/

  22. static 关键字有什么作用？

    static 代表 “静态” 的意思，可以用来修饰内部类、类的成员方法、类的成员变量、以及 static 静态代码块。另外还可以用来静态导包：静态导包就是 Java 包的静态导入，用import static 代替import，静态导包是JDK1.5的特性。

    静态导包的用法：

    • 一般的导入一个类是 import com.xxx.xxx.ClassName,而静态导包可以这样 import static com.xxx.xxx.ClassName.* 类名后面多了一个 .* 的意思是导入这个类的所有静态方法，如果只是导入某个静态方法的话就把 * 换成具体的类名

  23. static 变量和普通变量的区别？

    1. 所属目标不同

      静态变量属于类的变量，普通的变量属于对象的变量。

    2. 存储区域不同

      静态变量存储在方法区的静态区，普通变量存储再堆区。

    3. 加载时间不同

      • 静态变量是随着类的加载而加载的，随着类的消失而消失
      • 普通变量随着对象的加载而加载，随着对象的消失而消失

    4. 调用方式不同

      静态变量只能通过类名调用，普通变量只能通过对象调用。

  24. static 可以修饰局部变量么？

    不能修饰局部变量。局部变量属于方法，生命周期随着方法的调用而结束。

  25. final 关键字有哪些用法？

    • 修饰类：表示该类不能被继承
    • 修饰方法：表示该方法不能被重写
    • 修饰变量：表示该变量是一个常量，只能赋值一次，不能被修改

  26. final、finally、finalize 有什么区别？

    • final：是修饰符，如果修饰类，此类不能被继承；如果修饰方法和变量，则表示此方法不能被重写此变量不能再被改变，只能使用。
    • finally：是try{} catch{} finally{} 最后一部分，表示不论发生任何情况都会执行，finally 部分可以省略，但是如果 finally 部分存在，则一定会执行 finally 里面的代码。
    • finalize：是 Object 类的一个方法，在垃圾收集器执行的时候会调用被回收对象的此方法。

  28. 为什么 byte 取值范围为 -128 ~ 127？

    Java 设计 byte 类型为 1 个字节， 1 个字节占 8 位，即 8bit。计算机系统中是补码来存储的，首位为 0 表示正数，首位为 1 表示负数，所以：

    最大的补码用二进制表示为：01111111 = 127

    最小的补码用二进制表示为：10000000 = -128

  29. char 类型可以存储中文汉字吗？

    char 类型可以存储中文汉字。因为 Java 中使用的编码是 Unicode，一个 char类型占用 2 个字节，而一个中文也是占用 2 个字节，所以存放一个中文是没有问题的。

  30. 重载和重写有什么区别？

    重载 (Overload):

    在一个类中，具有相同的方法名，但是方法的参数不同，返回值类型可以相同也可以不同。

    • 被重载的方法必须改变参数列表（参数个数或参数类型不一样）；
    • 被重载的方法可以改变返回值类型；
    • 被重载的方法可以改变访问修饰符；
    • 被重载的方法可以声明新的或者更广的检查异常；
    • 无法以返回值类型作为重载方法的的区分标准；

    重写（override）：

    一般都是表示子类和父类之间的关系，其主要的特征是：方法名相同，参数相同，但是具体的实现不同。

    • 参数列表与被重写的方法的参数列表必须完全相同。
    • 返回类型与被重写方法的返回类型可以不相同，但是必须是父类返回值的派生类（Java1.5 及更早的版本返回类型要一样，Java7 及更高版本可以不同）。
    • 访问权限不能比父类中被重写的方法的访问权限更低。例如：如果父类的一个方法被声明为 public ,那么在子类中重写该方法就不能声明为 protected.
    • 父类的成员方法只能被它的子类重写。
    • 声明为 final 的方法不能被重写。
    • 声明为 static 的方法不能被重写，但是能够被再次声明。
    • 子类和父类在同一个包中，那么子类可以重写父类所有的方法，除了声明为 private 和 final 的方法。
    • 子类和父类不在同一个包中，那么子类只能够重写父类的得声明为 public 、protected 的非 final方法。
    • 重写的方法能够抛出任何非强制异常，无论被重写的方法是否抛出异常。但是，重写的方法不能抛出新的强制性异常，或者比被重写方法声明更广泛的强制性异常，反之则可以。
    • 构造方法不能被重写。
    • 如果不能继承一个类，则不能重写该类的方法。
    区别点 重载方法 重写方法
    发生范围 同一个类 子类
    参数列表 必须修改 一定不能修改
    返回值类型 可以修改 子类方法返回值类型比父类方法返回值类型更小或者相等
    异常 可修改 子类方法声明抛出的异常类应比父类方法声明抛出的异常类更小或者相等
    访问修饰符 可修改 一定不能做更严格的限制（可以降低限制）
    发生阶段 编译期 运行期

  31. 构造器可以被重写和重载吗？

    一个类的构造器不能被继承，所以它不能被重写。

    一个类里面可以有多个构造器，所以它可以被重载。

  32. main 方法可以被重写和重载吗？

    可以重载，但是 JVM 将始终调用原始的 main 方法，而永远不会调用重载的 main 方法。

    代码示例

    public static void main(String[] args) {
    System.out.println("这个是 main 方法");
}
public static void main() {
    System.out.println("这是重载的 main 方法");
}

    不能重写，因为 main 方法是静态方法，而 static 方法在 Java 中不能被覆盖。

  33. 私有方法能被重载或者重写吗？

    可以重载，不能重写。

  34. Java 中断言（assert）是什么？

    在 Java 中，断言（assert）关键字是从 Java1.4 引入的，断言是为了方便调试程序。为了避免和老版本的 Java 代码中使用了 assert 关键字导致错误，Java 在执行的时候默认是不启动断言检查的，如果要开启断言检查，需要用 -ea 来开启。

    在 idea 开启断言

    代码示例及效果

    public class AssertTest {
    public static void main(String[] args) {
        assert (1 == 0) : "1 == 0 是不成立的";
    }
}

    语法：

    • assert(boolean 表达式)

      如果 boolean表达式为 true，则程序继续执行。

      如果为 false，则程序抛出 AssertionError，并终止执行。

    • assert(boolean 表达式) : 错误信息

      如果 boolean表达式为 true，则程序继续执行。

      如果为 false ，则程序抛出AssertionError，并输出”错误信息“。

  35. Java 异常有哪些分类？

    下图是 Java 异常类的组织结构，红色区域的异常类表示是程序需要显示捕捉或者抛出的。

    • Throwable

      Throwable 是 Java 异常的顶级类，所有的异常都继承于这个类。Error，Exception 是异常类的两个大类。

    • Error

      Error 是非程序异常，即程序不能捕获的异常，一般是编译或者系统性的错误，如 OutOfMemorry 内存溢出异常。

    • Exception

      Exception 是程序异常类，由程序内部产生。Exception又分为运行时异常、非运行时异常。

      • 运行时异常

        运行时异常的特点是 Java 编译器不会检查它，也就是说，当程序中可能出现这类异常，即使没有用 try-catch 语句捕获它，也没有用 throws 子句声明抛出它，也会编译通过，运行时异常可处理或者不处理。运行时异常一般常用来定义系统的自定义异常，业务根据自定义异常做出不同的处理。

        常见的运行时异常如：NullPointException、ArrayIndexOutOfBoundsException 等。

      • 非运行时异常

        非运行时异常是程序必须处理的异常，捕获或者抛出，如果不处理程序就不能编译通过。如常见的 IOException、ClassNotFoundException 等。

  36. Error 和 Exception 有什么区别？

    Error 和 Exception 都属于异常总父类 Throwable的子类：

    • Error

      表示系统级的错误和程序不必处理的异常，一般会出现在运行期间的异常，如内存溢出，不可能指望程序能处理这样的情况。

    • Exception

      表示需要捕捉或者需要程序进行处理的异常，是一种设计或者实现问题，业绩上说，它表示如果程序运行正常，就不会发生异常。

  37. Java 中常见的异常有哪些？

    异常是 Java 程序中经常遇到的问题，一个异常就是一个 bug 需要花时间来定位异常问题。

    • NullPointerException

      空指针异常，操作一个 null 对象的方法或者属性时就会抛出这个异常。

    • OutOfMemoryError

      内存异常，这个不是程序可以控制的，是指要分配的对象的内存超出了当前醉的堆内存，需要调整堆内存大小（-Xmx）以及优化程序。

    • IOException

      IO 即：input，output。在读写磁盘文件、网络内容的时候经常会发生的一种异常，这种异常是受检查异常，需要进行手工捕获。

    • FileNotFoundException

      文件找不到异常，如果文件不存在就会抛出这种异常。

    • ClassNotFoundException

      类找不到异常，Java 开发中经常遇到，这是在加载类的时候抛出来的，即在类路径下不能加载指定的类。它是受检查异常，需要进行捕获。

    • ClassCastException

      类转换异常，将一个不是该类的实例转换成这个类就会抛出这个异常。这个是运行时异常，不需要捕获。

    • NoSuchMethodException

      没有这个方法异常，一般发生在反射调用方法的时候，它是受检查异常，需要进行捕获。

    • IndexOutOfBoundsException

      索引越界异常，当操作一个字符串或者数组的时候经常遇到的异常。

    • ArithmeticException

      算数异常，发生在数字的算术运算时的异常。

    • SQLException

      SQL 异常，发生在操作数据库时的异常。它是受检查异常需要进行捕获。

  38. Java 中常见的运行时异常有哪些？

    常见的运行时异常（RuntimeException）主要有以下几种：

    • NullPointerException: 空指针异常
    • IndexOutOfBoundsException：下表越界异常
    • NegativeArraySizeException：负数数组长度异常
    • ArithmeticException：算术异常
    • ClassNotFoundException: 类文件未找到异常
    • ClassCastException：类型强制转换异常
    • SecurityException：违背安全原则异常

  39. 运行时异常与受检查异常有什么区别？

    运行时异常：

    表示虚拟机的通常操作中可能遇到的异常，是一种常见的运行错误，只要程序设计的没有问题通常就不会发生。

    受检查异常：

    跟程序运行的上下文环境有关，即使程序设计无误，仍有可能因为使用的问题而引发。

    区别：

    Java 编译器要求方法必须用（try-catch）声明抛出可能发生的受检查异常，但是并不要求必须声明爬出未被捕获的运行时异常。

  40. 什么时候会发生空指针异常？

    当一个变量的值为 null 时，在 Java 里面表示一个不存在的空对象，没有实际内容，没有给它分配内存，null 也是对象成员变量的默认值。所以，一个对象如果没有进行初始化操作，这时调用这个对象的方法或者变量，就会出现空指针异常。

    从类结构图来看，空指针异常属于运行时异常 RuntimeException 的子类，无需手动捕获，只有在程序运行时才可能报出来，并且会造成程序中断。

  41. 知道哪些可以避免空指针的方法？

    1. 字符串比较，常量放在前面

      if (SUCCESS.equals(employee.status)) {
    //TODO
}

    2. 初始化默认值

      在给对象初始化的时候就给它一个默认值

    3. 返回空集合

      在返回一个集合的时候，默认会是一个 null ，统一规范返回一个空集合。如：

      public List getList() {
    List list = Test.getOneList();
    return list == null ? new ArrayList() : list;
}

    4. 断言

      断言是用来检查程序的安全性的，在使用之前进行条件检查，如果不符合就报异常，符合就继续

      Java 中自带的断言关键字：assert , 如：

      Employee employee = null;
assert (employee != null) : "employee 不可以为 null";

      Spring 中的用法

      Assert.notNull(employee, "employee 不能为空")

  42. throw 和 throws 的区别？

    • throw：是真实抛出一个异常
    • throws: 是声明可能会抛出一个异常

  43. try-catch-finally 中哪个部分可以省略？

    try -catch-finally 其中 catch 和 finally 都可以被省略，但是不能同时省略，也就是说 try 的时候，必须后面跟一个 catch 和 finally。

  44. Java 可以一次 catch 多个异常吗？

    Java1.7 开始就可以一次 catch多个异常，比如：

    try {
    System.out.println(11);
}catch (NullPointerException | ClassCastException e) {
    e.getMessage();
}

  45. try 里面 return，finally 还会执行吗？

    1. 不管 try 有没有 return，finally 都会执行
    2. 在 try 中 return，在 finally 执行前会把结果保存起来，即使在 finally 中有修改也会以 try 中保存的值为准，但如果是引用类型，修改的属性会以 finally 修改后的为准
    3. 如果 try/finally 都有return, 直接返回 finally 中的 return
