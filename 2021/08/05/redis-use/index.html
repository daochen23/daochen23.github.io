Redis中5大常用类型及命令

发表于2021-08-05|更新于2021-08-09|Redis
|字数总计:1.9k|阅读时长:7分钟|阅读量:

key键操作

  • 查看当前库中所有的key

    keys *

  • 判断某个key是否存在

    exists key

  • 查看某个key的类型

    type key

  • 删除某个key

    del key

  • 异步非阻塞删除

    unlink key

    和dle key区别是，unlink只是将key从keyspace元数据中删除，真正的删除是在后续异步操作。

  • 设置key的过期时间

    expire key 10

  • 查看key还剩多少时间过期

    ttl key

    -1 表示永不过期；-2 表示已经过期

  • 切换数据库

    select 1

    redis一共有16个数据库，0-15默认使用第一个数据库

  • 查看当前数据库的key数量

    dbsize

  • 清空当前数据库

    flushdb

  • 清空所有的数据库

    flusha

string（Redis字符串）

string类型是二进制安全的。意味着Redis的string可以包含任何数据。比如jpg图片或者序列化对象。一个string的value值最大为512M

常用命令

  • 设置一个string类型key

    set <key> <value>

    如果设置的key已经存在那么新的value值会覆盖原来的value值，如：set k1 v1,这时的k1的值为v1,set k1 v2,那么这个时候原本的v1会被v2覆盖

  • 获取key的值

    get <key>

  • 在value后面追加新的数据

    append <key> <value>

  • 查看value值的长度

    strlen <key>

  • 设置一个string类型的key

    setnx <key> <value>

    和set不同的是只有当key不存在时，才可以设定，常用来作为分布式锁

  • 自增

    incr <key>

    当key存储的值为数字时，自增1.如果是一个不存在的key,则初始化的值为1

  • 自减

    decr <key>

    效果与incr相反

  • 自定义自增数量

    incrby <key> <步长>

    步长可以为负数，如果为负数效果就是自减

  • 自定义自减数量

    decr <key> <步长>

    步长可以为负数，如果为负数效果就是自增

  • 批量设置string类型的key

    mset <key1> <value1> <key2> <value2>

  • 批量获取string类型的key

    mget <key1> <key2>

  • 批量设置不存在的string类型的key

    msetnx <key1> <value1> <key2> <value2>

    以上的批量设置是一个原子性操作，有一个失败都失败

  • 获取某个范围的value值

    getrange <key> <开始下标> <结束下标>

  • 覆盖某个范围的value值

    setrange <key> <下标> <value>

  • 设置一个包含过期时间的key

    setex <key> <过期时间> <value>

  • 新值换旧值

    getset <key> <value>

    如果k1的值为100，那么如果使用上面的命令100会被新的value值覆盖掉

list(Redis列表)

Redis列表是简单的字符串列表，按照顺序插入，可以添加一个元素到列表头部（左边）或者尾部（右边）。它的底层实际上一个双向链表，对两端的操作性能很高，通过索引下标操作中间节点性能较差。

常用命令

  • 创建一个列表

    lpush/rpush <key> <value1> <value2> <value3>

  • 从列表中取出一个值

    lpop/rpop <key>

    当列表中一个值都没有的时候，这个列表就不存在了

  • 从列表右边取出一个值放到另外一个列表左边

    rpoplpush <key1> <key2>

  • 从左到右获取列表中的值

    lrange <key> <开始下标> <结束下标>

  • 从左到右获取索引下标的元素

    lindex <key> <下标>

  • 获取列表的长度

    llen <key>

  • 在指定元素前面或者后面添加元素

    linsert <key> <after>/<before> <指定元素> <添加元素>

  • 从左边删除n个相同的元素

    lrem <key> <n> <value>

  • 替换指定位置的元素

    lset <key> <index> <value>

set(Redis集合)

Redis set对外提供的功能与list类似是一个列表的功能，特殊之处在于set是可以自动排重，当你需要存储一个列表数据，又不希望出现重复数据时，set是一个很好的选择

  • 添加一个元素到集合中

    sadd <key> <value1> <value2>

    如果添加的元素已经在集合中存在，那么这个元素就不会被重复添加

  • 查看集合中的所有元素

    smembers <key>

  • 判断集合中是否包含某个元素

    sismember <key> <value>

    如果存在返回1，否则返回0

  • 查看集合中元素的个数

    scard <key>

  • 删除集合中的某些元素

    srem <key> <value1> <value2>

  • 随机从集合中拿出一个值

    spop <key>

    拿出来的值会从集合中删除

  • 随机从集合中取出N个值

    srandmember <key> <count>

    出去来的值不会从集合中删除

  • 把集合中的值移动到另一个集合中

    smove <原集合> <目标集合> <需要移动的数据>

  • 两个集合的交集

    sinter <key1> <key2>

  • 两个集合的并集

    sunion <key1> <key2>

  • 返回两个集合的差集

    sdiff <key1> <key2>

    返回的结果是在key1存在，key2不存在的数据

hash(Redis哈希)

Redis hash是一个键值对集合。是一个string类型的field和value的映射表，hash特别适合用来存储对象。

常用命令

  • 设置一个hash类型的key

    hset <key> <field1> <value1> <field2> <value2>

  • 获取key中某个字段的值

    hget <key> <field>

  • 批量设置hash的值

    hmset <key> <field1> <value1> <field2> <value2>

  • 查看key中是否包含某个field

    hexists <key> <field>

  • 获取key中所有的field

    hkeys <key>

  • 获取key中所有field对应的值

    hvals <key>

  • 为field所对应的值自增或者自减

    hincrby <key> <field> <自增多少或者自减多少>

  • 为key中的field设置值

    hsetnx <key> <field> <value>

    需要保证在该key中这个field不能存在，否则无法设置成功

score set(Redis有序集合)

和set不同之处：有序集合的每个成员都关联了一个评分（权重score），这个评分（score）被用来按照从低到高的方式排列。集合中的成员是唯一的，但是评分是可以重复的

  • 创建一个score set集合

    zadd <key> <score1> <value1> <score2> <value2>

    元素可以是一个也可以是多个

  • 返回集合内容

    zrange <key> <开始下标> <结束下标> [withscores]

    返回指定索引范围内的内容，如果加上了withscore那么value对应的score也会返回(默认按照从小到大的顺序)

  • 返回score范围内的元素

    zrangebyscore <key> <min> <max> [withscores]

  • 返回score范围内的元素

    zrevrangebyscore <key> <max> <min> [withscores]

    与zrangebyscore效果相反

  • 修改（自增/自减）指定元素的score

    zincrby <key> <自增/自减数量> <value>

  • 删除集合指定元素

    zrem <key> <value>

  • 统计集合中score区间内元素个数

    zcount <key> <min> <max>

  • 返回元素在集合中的排名

    zrank <key> <value>

    返回的排名是从0开始的（从小到大排列）
